Central de Controle do ATLAS

Este índice é o ponto de entrada do ecossistema ATLAS. Tudo que é serviço crítico roda em containers, com domínios separados por responsabilidade: memória (MNEMOS), execução (FORGE), eventos (PYTHIA) e observabilidade (ORACULO). Passe o mouse nos projetos para ver o que cada um faz.

Host
Windows • Docker • WSL2
Operação 24/7 • acesso remoto via RDP/SSH
Regra de Ouro
Nada crítico fora de container
Infra reproduzível, versionada e documentada.
Interação futura
Wake word: “Atlas”
Modelo de comandos auditáveis (voz → intenção → ação).

MNEMOS — memória viva

Domínio responsável por persistência e conhecimento. Ideal para bancos e catálogo do acervo.

  • PostgreSQL + dados estruturados
  • Inventário / indexação do acervo
  • Metadados (onde está, versão, hash, origem)

FORGE — execução

Domínio que roda tarefas reexecutáveis e pipelines. Lugar natural do CI pessoal.

  • Jenkins / Portainer
  • Jobs, runners e schedules
  • Backup e versionador integrados

PYTHIA — eventos & API

Domínio que reage a eventos e coordena decisões. Integra Python, n8n e webhooks.

  • API de comunicação entre serviços
  • Orquestração de fluxos
  • Disparo de ações e alertas

ORACULO — dashboard

Domínio visual: relatórios, cadastros e visões de status do ecossistema.

  • Painel web (consulta e controle)
  • Relatórios e históricos
  • Visão de saúde e execução
